בפרק הזה ביומן שלי אכתוב על מסעותיי ברחבי רשת I2P. הכתבה הזאת היא חלק מסדרת כתבות בנושא ה-deep web. ההקדמה נמצאת בקישור הבא: הצד האחר של הרשת - הקדמה

I2P

הצד האחר של הרשת - הקדמה

את ה-I2P אני מריץ על docker. הנה התמונה שמצאתי ב-Docker Hub:

Docker Hub

https://hub.docker.com/r/ksey/i2pd

https://hub.docker.com/r/ksey/i2pd

משכתי את התמונה:

$ docker pull ksey/i2pd

יצרתי תיקיה חדשה למידע שעתיד לרדת מהרשת ונכנסתי לתיקיה:

$ mkdir i2pd && cd i2pd

יצרתי שני קבצי קונפיגורציה, אחד כללי ואחד ל-tunnels:

$ touch i2pd.conf $ touch tunnels.conf

ערכתי אותם לפי הדוגמאות האלה:

https://github.com/PurpleI2P/i2pd/blob/openssl/contrib/i2pd.conf https://github.com/PurpleI2P/i2pd/blob/openssl/contrib/tunnels.conf

https://github.com/PurpleI2P/i2pd/blob/openssl/contrib/i2pd.conf

https://github.com/PurpleI2P/i2pd/blob/openssl/contrib/tunnels.conf

הרצתי קונטיינר שירוץ ברקע ויימחק ברגע שיעצור. קישרתי את קבצי הקונפיגורציה לתיקיה המיועדת לכך בקונטיינר ופתחתי את הפורטים הרצויים (על-פי ההוראות באתר):

$ docker run -d --rm --name i2pd -d -p 7070:7070 -p 4444:4444 -p 4447:4447 -p 7656:7656 -v /$PWD:/var/lib/i2pd ksey/i2pd

נכנסתי ל-localhost פורט 7070 בדפדפן (Firefox ב-incognito):

לאחר מכן הורדתי chromium, אותו קינפגתי וכיביתי כמעט כל אפשרות שקיימת בדפדפן. אני מריץ אותו בצורה הבאה:

$ chromium-browser --proxy-server="http://127.0.0.1:4444" --incognito

בהמשך חזרתי ל-Firefox מוקשח.

חיפשתי הסברים ברשת והגעתי לכתובת הבאה אותה הזנתי בדפדפן:

http://identiguy.i2p/

http://identiguy.i2p/

למען האמת אני לא כל כך יודע מה אני עושה. לא קראתי מספיק ואני פשוט מנסה להגיע לאתרים. לחצתי על כמה קישורים, רובם מובילים לדף הבא:

מבחר אתרים מעניינים שמצאתי. אגב, במושגים של I2P אתרים נקראים eepsites.

שער כניסה ל-Telegram על גבי I2P
